# Architecture

This project is a patient-simulation QA harness for the Pretty Good AI voice challenge. It places guarded outbound calls through Twilio to the single allowed assessment number, serves signed Twilio webhooks from a FastAPI app, connects the call to a bidirectional Twilio Media Stream, and bridges audio to an OpenAI Realtime patient persona. Each scenario is loaded from YAML, giving the live caller a stable identity, objective, steering ladder, expected behavior, and stop condition instead of a one-question script.

The system is designed as an evidence factory rather than a one-off caller. Twilio records the full call as MP3, the local server writes Twilio/OpenAI event logs, Groq transcribes the final recording, and the final package links recordings, transcripts, scenarios, event logs, and curated bugs. The main design choices are speed and proof: Realtime speech-to-speech avoids a slower STT-LLM-TTS loop during calls, post-call Groq transcription produces reviewable evidence, the dialer refuses any number except `+18054398008`, public webhooks require Twilio signatures by default, stream metadata is signed, and all project state stays local to this repository.
